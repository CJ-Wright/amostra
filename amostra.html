amostra.ignition module¶

amostra.ignition.start_server(config=None)¶

Amostra service startup script. Returns tornado event loop provided configuration.

Parameters:config (dict) – Command line arguments always have priority over local config or yaml files.Using these parameters, a tornado event loop is created. Keep in mind that this server is started in lazy fashion. It does not verify the existence of a mongo instance running on the specified location.
